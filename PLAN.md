# Менторская программа на 5–6 месяцев: Junior AI Backend

## Summary
Я беру роль ментора по треку `Python -> FastAPI -> WebSocket/SSE -> PostgreSQL -> SQLAlchemy -> Alembic -> Redis -> Docker -> OpenAI API -> RAG -> pgvector -> production basics`.

Формат работы:
- Темп: `15–20 часов в неделю`
- Режим: `ежедневный трек` с `сбалансированным` контролем
- Главная цель: выйти на уровень `Junior backend/AI` и собрать портфолио под первую работу
- Канал трекинга: `прямо в этом чате`
- Язык: объяснения и разборы на русском, но с постепенным встраиванием английского в обучение

Моя роль:
- разбивать путь на короткие этапы и ежедневные задачи
- объяснять темы простым языком
- проверять прогресс, домашки, проекты и пробелы
- корректировать темп, если ты проседаешь или идешь быстрее
- готовить тебя не только “учиться”, но и к собеседованиям, портфолио и рабочему мышлению

## Как будет устроено менторство
### Рабочий цикл
- Каждый день я выдаю тебе 1 учебный блок: теория, практика, мини-задание, критерии готовности.
- Ты присылаешь сюда отчет: что сделал, что не понял, где застрял, что получилось.
- Я отвечаю разбором: ошибки, уточнения, как улучшить, что делать следующим шагом.
- Каждую неделю я делаю ревью недели: прогресс, слабые места, что закрепить, что убрать или ускорить.

### Формат ежедневного задания
Каждый день я даю:
- цель дня
- 1 основную тему
- 1 практическое упражнение
- 1 мини-задачу на код
- 3–7 терминов/слов на английском по теме
- критерий “день закрыт, если…”

### Формат недельного спринта
Каждая неделя содержит:
- ключевой результат недели
- список тем
- практику
- мини-проект или часть проекта
- чек по английскому
- контрольный вопросник
- итоговый артефакт в портфолио или git-репозиторий

## План по месяцам
### Месяц 1: Python Core + инженерная база
Цель:
- уверенно писать базовый Python и понимать, как устроен нормальный проект

Что проходим:
- синтаксис, коллекции, функции, циклы, модули
- OOP, исключения, файлы, JSON
- `venv`, зависимости, структура проекта
- основы `typing`
- основы `async` на интуитивном уровне
- git-база для учебных проектов

Практика:
- маленькие консольные задачи
- парсер/обработчик JSON
- мини CLI-проект
- небольшой HTTP-клиент на Python

Результат:
- 2–3 маленьких проекта
- уверенная база для перехода в backend

### Месяц 2: FastAPI и backend-мышление
Цель:
- научиться строить API и понимать веб-базу

Что проходим:
- HTTP, REST, JSON, статус-коды
- REST APIs и JSON APIs как основной формат backend-интерфейсов
- FastAPI, роуты, Pydantic, валидация
- структура backend-проекта
- конфиги, `.env`, ошибки, middleware basics
- auth basics, JWT на базовом уровне
- Basic Authentication, Token Authentication, Cookie Based Auth
- OAuth basics и отличие OAuth от обычной token auth
- WebSocket basics для real-time сценариев
- SSE basics для stream-ответов и событий сервера

Практика:
- CRUD API
- валидация входящих данных
- фильтры, пагинация, схемы ответа
- auth-практика: JWT, cookies, bearer token, basic auth
- простой WebSocket endpoint
- простой SSE endpoint для потоковой отдачи событий

Результат:
- первый полноценный backend-сервис на FastAPI

### Месяц 3: PostgreSQL + SQLAlchemy + Alembic
Цель:
- перейти к реальному backend с БД и миграциями

Что проходим:
- SQL база: select, joins, group by, indexes
- PostgreSQL как рабочая БД
- SQLAlchemy ORM/Core basics
- связи моделей
- transactions: commit, rollback, atomic operations
- Alembic migrations
- database indexes и базовая оптимизация запросов
- scaling databases: read replicas, connection pooling, partitioning basics
- репозиторный и сервисный слой на базовом уровне

Практика:
- перенос CRUD API на PostgreSQL
- создание схем, миграций, связей
- миграции схемы через Alembic в реальном проекте
- транзакции для нескольких связанных операций
- добавление индексов и проверка эффекта через `EXPLAIN`
- несколько реальных запросов и фильтров

Результат:
- production-like API с нормальной БД

### Месяц 4: Redis + Docker + фоновые процессы
Цель:
- приблизить стек к реальному production backend

Что проходим:
- Redis как cache
- Redis как broker для Celery
- TTL, кеширование, типовые сценарии
- Celery workers для фоновых и параллельных задач
- retries, task status и базовая устойчивость задач
- RabbitMQ basics: exchanges, queues, routing keys, producers, consumers
- message-driven подход и отличие RabbitMQ от Celery/Redis task queue
- фоновые задачи и базовая оркестрация worker-процессов
- WebSocket + Redis pub/sub для live-уведомлений
- SSE для прогресса фоновых задач, логов и AI streaming
- Docker, Docker Compose
- Nginx basics: reverse proxy, static files, upstream services
- локальный запуск нескольких сервисов вместе

Практика:
- API + PostgreSQL + Redis
- кэширование ответов
- Celery + Redis для параллельных задач
- RabbitMQ producer/consumer для событий приложения
- простой message pipeline через RabbitMQ
- фоновые задачи на отправку/обработку
- task status endpoint для отслеживания задач
- real-time notifications через WebSocket
- progress updates через SSE
- упаковка всего в Docker Compose
- Nginx reverse proxy перед API

Результат:
- полноценный сервис из нескольких компонентов

### Месяц 5: OpenAI API + RAG + pgvector
Цель:
- собрать AI backend и понять практический RAG

Что проходим:
- OpenAI API basics
- prompts, tokens, embeddings
- RAG pipeline
- chunking, retrieval
- `pgvector` в PostgreSQL
- FastAPI endpoint для AI-функционала

Практика:
- ingestion документов
- хранение embeddings
- semantic search
- AI assistant по своей базе знаний

Результат:
- рабочий AI backend-проект для портфолио

### Месяц 6: Production readiness + трудоустройство
Цель:
- упаковать все в резюме-готовый и interview-ready вид

Что проходим:
- тесты: unit/integration/API
- логирование, мониторинг basics
- ошибки, устойчивость, безопасность basics
- architectural patterns: layered architecture, clean/service/repository подход
- monolithic apps и когда монолит лучше микросервисов
- microservices basics и риски преждевременного разбиения
- GitHub Actions CI/CD: workflows, jobs, runners
- CI для тестов, линтинга и проверки форматирования
- CD basics: Docker build, registry, deploy pipeline
- README, архитектурное описание, демо
- разбор собеседовательных вопросов
- портфолио, резюме, позиционирование

Практика:
- доведение главного проекта
- полировка структуры репозитория
- архитектурная схема проекта: API, DB, cache, workers, queue, proxy
- сравнение monolith vs microservices на своем проекте
- настройка `.github/workflows/ci.yml`
- pipeline для тестов, сборки Docker image и базового деплоя
- подготовка легенды проекта: что сделал, какие проблемы решал, почему так спроектировал

Результат:
- 1 главный сильный проект
- 2–3 вспомогательных проекта
- база под отклики и техсобесы

## Как встроим английский без перегруза
Подход:
- не учить английский отдельно большим блоком
- встраивать его в каждый день маленькими дозами

Правила:
- каждый день 3–7 терминов по текущей теме
- читать названия методов, ошибок, документации в оригинале
- коротко переводить смысл своими словами
- раз в неделю делать мини-разбор англоязычной документации или статьи
- не требовать свободного английского сразу; цель на 5–6 месяцев — начать уверенно читать документацию и понимать базовый технический контекст

## Test Plan
Проверка прогресса будет идти так:
- ежедневно: закрыл ли цель дня и можешь ли объяснить тему своими словами
- еженедельно: есть ли код, мини-результат и понимание ключевых концепций
- ежемесячно: есть ли завершенный модуль и проектный артефакт
- в конце: можешь ли ты собрать, объяснить и защитить свой AI backend проект как junior-кандидат

Контрольные сценарии:
- можешь написать и запустить код без постоянных подсказок
- понимаешь, как API связано с БД, кэшем и AI-слоем
- можешь объяснить, зачем нужен каждый элемент стека со скрина
- можешь пройти базовое техническое интервью по своему стеку
- можешь показать 1–2 проекта без ощущения “это просто учебка”

## Assumptions
- Начинаем с уровня `новичок`, поэтому фундамент важнее скорости.
- Основной трек делаем под `Junior backend/AI`, а не под чистый data science.
- Обучение и трекинг ведем в этом чате.
- Объяснения я даю на русском, но с регулярным и мягким встраиванием английского.
- Основной проектный фокус — `AI backend`, а не просто классический CRUD.
- Если темп `15–20 часов` станет слишком тяжелым, я уменьшу недельный объем, но не сломаю общую последовательность.

Следующий рабочий формат в рамках этого плана:
- ты пишешь мне `День 1`
- я выдаю тебе первый полноценный менторский день: цель, теория, практика, мини-задачу, английский блок и критерий готовности
